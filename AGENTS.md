# AGENTS.md

## Global Rules for This Repository

1. **Always commit .gitignore first.**
   - If a .gitignore file is created or updated, it must be committed before any other changes. This ensures sensitive files (like .env) and system clutter are never accidentally tracked.

2. **If the user says to commit, confirm whether only local or also remote.**
   - Before running a commit, always clarify if the user wants the changes just saved locally or also pushed to the remote repository (GitHub/cloud).

3. **When doing add, separate into logical segments.**
   - When staging files (git add), group them by logical units: a set of related tasks, a single folder, or a historical batch. This makes it easy to trace changes and roll back if needed.

4. **Most important: the user is an author working on novel-writing projects.**
   - Always orient your actions and explanations toward supporting novel-writing, not just software development. Use plain language to explain what you’re doing, focusing on how it helps the author’s writing workflow and project organization.

---

These rules apply at all times for all agents and automation in this repository.
